🎯 Interview Coach

基于 Harness Engineering（驾驭工程） 构建的 AI 面试深度解答 Agent — 输入面试题，自动生成含记忆法、原理拆解、答题思路的完整文档，一键推送到 GitHub

⚙️ Harness Engineering 🤖 多 Agent 协作 🧠 Claude Code Skill

🏗️ 工作流程

📥 阶段1: 题目解析
拆分面试题为独立题目，识别类型
↓
🤖 Q1 答题
interview-answerer
🤖 Q2 答题
interview-answerer
🤖 Q3 答题
interview-answerer
↓
✅ Q1 审查
quality-reviewer
✅ Q2 审查
quality-reviewer
✅ Q3 审查
quality-reviewer
↓ (不合格自动重答，最多2次)
📄 阶段4: 文档组装
doc-assembler — 生成完整 Markdown 文档
↓
🚀 阶段5-6: 落盘 & 推送
Write 到 outputs/ → git commit → git push → GitHub

🧩 每道题的回答结构

🧠 联想记忆法

最先给出，帮你快速编码到长期记忆

  • 记忆口诀/场景联想
  • 记忆原理（认知钩子）
  • 关联知识（知识锚定）

📖 深度解答

按四层递进拆解原理

  • 核心概念（是什么）
  • 底层原理（为什么）
  • 实践应用（怎么用）
  • 深入思考（注意事项）

🗺️ 回答思路

面试现场怎么答、怎么得分

  • 答题逻辑框架
  • 重点得分点
  • 常见误区 & 过渡话术

⚙️ Harness Engineering 体现

驾驭工程维度本项目实现
多 Agent 协作3 个专用 Agent（解题者 + 审查者 + 文档者），各司其职
Workflow 编排确定性 6 阶段流水线，每阶段有明确的输入输出契约
质量门控审查 Agent 逐项检查 15 条标准，不合格自动返工重答
Skill 封装/面经助手 一行命令触发全流程
自动部署文档生成后自动 git commit + push 到 GitHub
可追溯性每步输出落盘，Git 历史追踪每次解答

🚀 如何使用

# 克隆仓库
git clone git@github.com:Dddddduo/dduo-interview-coach.git
cd dduo-interview-coach

# 在 Claude Code 中打开此目录
claude

# 输入 /面经助手 + 你的面试题
/面经助手
第1题：请解释MySQL的索引底层数据结构，为什么选用B+树？
第2题：Redis的过期策略有哪些？如何保证缓存与数据库一致性？
第3题：描述你项目中遇到的最大技术挑战及解决方案
📄 查看示例输出 🔗 GitHub 仓库

🤖 Agent 体系

Agent模型职责
interview-answererOpus深度解答单道面试题，产出记忆法 + 原理 + 思路
quality-reviewerSonnet15 项清单逐项审查，PASS/FAIL + 具体修改建议
doc-assemblerSonnet组装最终文档，格式化、目录、元信息
