# 怎么保证数据库和缓存一致性？

推荐延迟双删（Delayed Double Delete）：先更新DB→删缓存→延迟500ms再删一次。为什么先DB后缓存？即使短暂不一致，下次读会从DB拉最新数据。其他方案：CDC监听binlog同步（Canal+MQ）。
