# 为什么默认是可重复读？

历史原因：MySQL 5.0之前RC+Statement格式binlog会导致主从复制数据不一致。RR解决的：脏读、不可重复读、幻读（Next-Key Lock）、主从复制一致性。MySQL 8.0默认binlog为ROW格式，理论上可改为RC默认。
