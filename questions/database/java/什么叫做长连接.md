# 什么叫做长连接？

长连接在同一个TCP连接上发送多个请求-响应（HTTP/1.1默认Keep-Alive）。优点：减少握手开销、避免TIME_WAIT。缺点：需心跳保活、占用资源。HTTP/2所有请求共享一个连接（多路复用）。
