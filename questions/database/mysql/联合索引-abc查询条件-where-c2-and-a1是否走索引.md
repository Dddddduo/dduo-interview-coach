# 联合索引 (a,b,c)，查询条件 where c=2 and a=1，是否走索引

---

### 🧠 联想记忆法

**记忆口诀/联想**: **"引擎自动调顺序，最左前缀是铁律"**

口诀前半句"引擎自动调顺序"对应 MySQL 优化器会自动重排 WHERE 条件（`c=2 AND a=1` → `a=1 AND c=2`），因此不要求 SQL 书写顺序与索引列顺序一致。口诀后半句"最左前缀是铁律"对应索引使用的根本原则：必须从最左列开始匹配，中间不能断。

**记忆原理**: 将"优化器自动性"和"最左前缀原则"这两个关键点压缩成一句对仗口诀，朗朗上口。"调顺序"和"是铁律"形成对比反差——优化器可以灵活调整 WHERE 顺序，但最左前缀原则是不可突破的底线。

**关联知识**: 类比字典查找——联合索引 (a, b, c) 就像一本电话簿先按姓氏(a)排、再按名字(b)排、最后按电话号码(c)排。查询时优化器会自动把你问的问题重新组织成最高效的查找顺序，但无论怎么组织，没有姓氏(a)就直接查名字(b)和电话(c)是找不到的。

### 📖 深度解答

**1. 核心概念（是什么）**

**直接答案：会走索引，但只使用了联合索引中的 a 列，c 列无法通过索引直接定位。**

具体来说：

| 索引使用情况 | 说明 |
|---|---|
| a 列 | **走索引** —— 通过 B+ 树定位到 a=1 的记录范围 |
| b 列 | **未使用** —— 条件中没有 b 的等值条件 |
| c 列 | **通过 ICP（Index Condition Pushdown，索引条件下推）或回表后过滤**，无法通过索引直接定位 |

MySQL 优化器（Optimizer）会自动将 `WHERE c=2 AND a=1` 重写为 `WHERE a=1 AND c=2`，因此 SQL 的书写顺序不影响索引使用。

**2. 底层原理（为什么）**

**2.1 联合索引的 B+ 树结构**

联合索引 (a, b, c) 的 B+ 树按照以下规则构建：

- **第一排序键**：按 a 列升序排列
- **第二排序键**：a 相同时，按 b 列升序排列
- **第三排序键**：a 和 b 都相同时，按 c 列升序排列

B+ 树的非叶子节点和叶子节点中存储的是 (a, b, c) 这个三元组，按 `(a, b, c)` 的字典序（lexicographical order）组织。

**2.2 最左前缀原则（Leftmost Prefix Principle）**

MySQL 使用联合索引时，只能从最左边开始连续匹配。索引 (a, b, c) 等价于创建了三个索引：`(a)`、`(a, b)`、`(a, b, c)`。

条件 `WHERE a=1 AND c=2` 中，a 匹配了最左前缀，b 缺失，因此 c 无法利用索引的有序性进行定位。这被称为**中间断裂（gap in the middle）**。

**2.3 查询优化器的自动重排**

MySQL 优化器将 `WHERE c=2 AND a=1` 解析后，由于 AND 是**可交换的（commutative）**，优化器可以自由重排为 `WHERE a=1 AND c=2`。完整步骤包括：词法分析、CNF 转换、条件简化、条件排序（基于索引定义）、访问路径选择。

**2.4 ICP（Index Condition Pushdown，索引条件下推）**

MySQL 5.6 引入。在 `WHERE a=1 AND c=2` 中，存储引擎通过索引找到 a=1 的所有记录后，**在索引层面**直接判断 c=2 的条件，只有满足条件的记录才回表。Extra 显示 `Using index condition`。

**2.5 key_len 计算规则**

| 类型 | 字节 | 备注 |
|---|---|---|
| INT | 4 | |
| VARCHAR(n) | `n * 字符集 + 1(可空) + 2(变长)` | |
| 可空列 | +1 | |

**3. 实践应用（怎么用）**

**场景对比表：**

| SQL | 走索引？ | key_len | Extra |
|-----|---------|---------|-------|
| `WHERE a=1 AND b=2 AND c=3` | 是 (a,b,c) | 12 | Using index |
| `WHERE c=2 AND a=1` | 是 (a列) | 4 | Using index condition |
| `WHERE b=1 AND a=2` | 是 (a,b列) | 8 | Using index condition |
| `WHERE b=1 AND c=2` | **否** | NULL | Using where |
| `WHERE a=1 AND c=2` | 是 (a列) | 4 | Using index condition |

**4. 深入思考（注意事项）**

常见误区：WHERE 条件必须按索引顺序写（错）、走了索引等于用了全部列（错）、ICP 可替代索引列（错）。

范围查询阻断：`WHERE a=1 AND b>10 AND c=5` 中 c 列无法走索引。
