# 进程和线程的区别是什么？请从资源分配和执行调度的角度分析

### 联想记忆法
**记忆口诀**: "进程是房子，线程是房间里的人"
**记忆原理**: 房子(进程)=独立的资源容器(地址空间、文件描述符)，人(线程)=执行单元，共享家具家电(内存/文件)
**关联知识**: Docker 容器（进程级轻量隔离）、Go 协程（用户态更轻执行单元）

### 深度解答

#### 核心概念（是什么）
- **进程**: 操作系统进行资源分配和调度的基本单位，拥有独立地址空间
- **线程**: CPU 调度和执行的最小单位，共享进程资源，独立栈和寄存器

一句话：**进程负责"拥有资源"，线程负责"执行任务"**

#### 底层原理（为什么）
Linux 内核中，进程和线程都用 `task_struct` 表示，区别在资源共享程度：
- `fork()` 创建进程 → 复制父进程地址空间（Copy-on-Write 优化）
- `clone(CLONE_VM | CLONE_FS | CLONE_FILES)` 创建线程 → 共享地址空间

关键区别维度：
| 维度 | 进程 | 线程 |
|------|------|------|
| 资源拥有 | 独立地址空间、文件表 | 共享进程资源 |
| 创建开销 | 大（PCB、页表、地址空间） | 小（TCB+栈） |
| 上下文切换 | 重（切换页表、刷新 TLB） | 轻（同进程不需切页表） |
| 通信方式 | IPC（管道/消息队列/共享内存） | 直接读写共享变量 |
| 隔离性 | 强（一个崩溃不影响其他） | 弱（可能带崩整个进程） |

#### 实践应用（怎么用）
- **Chrome 浏览器**: 多进程架构，一个 Tab 崩溃不影响其他
- **Nginx**: 多进程 Worker，稳定性优先
- **Java 应用服务器**: 多线程，共享连接池和缓存
- **微服务**: 多进程，独立部署和扩缩容

#### 深入思考（注意事项）
- 协程(Coroutine)比线程更轻：用户态调度、动态栈（几 KB vs 1-8MB）
- 执行单元演化趋势: 进程 → 线程 → 协程
- 多线程不一定比多进程快（CPU 密集且无共享需求时多进程更好）
- 线程有独立的栈、寄存器和 TLS（线程局部存储）
